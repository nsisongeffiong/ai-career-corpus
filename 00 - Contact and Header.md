# Contact and Header — Canonical

Use this verbatim for every generated resume, cover letter, and application. Do not pull contact details from old resumes — this file is the source of truth.

## Resume header

Name + up to two centered contact lines. **Each line is its own paragraph — never chain everything onto one line (it overflows and wraps).** See `09 - Resume Style Spec.md` for rendering.

**[YOUR NAME]**
Line 1 (always): `[phone] | [email] | [City, Region] | linkedin.com/in/[handle]`
Line 2 (engineering/technical/content roles only): `github.com/[handle] | [yoursite.com]`

For non-technical roles, omit line 2 — line 1 alone. LinkedIn always lives on line 1.

## Conventions

- File naming for generated resumes: `[Your_Name]_<Role>_Resume.docx`
- Cover letters: `[Your_Name]_<Role>_Cover_Letter.docx`
- Save each application's outputs (resume, cover letter, JD copy) to: `<your applications folder>/Per-Application/<Company - Role>/`
- Header on cover letters matches the resume header exactly
- Do not include date of birth, marital status, photo, or references on resumes (North American convention — adjust for your market)
- "References available upon request" line: omit (assumed)
